About TrinityTix

This project demonstrates the ability to architect a full-stack system utilizing three distinct database paradigms:

  • Relational (SQL): SQLite
  • In-Memory (Key-Value): Redis
  • NoSQL (Document): MongoDB

The core concept showcased here is Eventual Consistency across distributed systems. By leveraging an Operation Log (Oplog) with strict First-Come-First-Served timestamp comparisons and status priority levels (Booked > Held > Available), the systems can independently process user requests and synchronize to a unified, conflict-free state without locking.
